Overview

A Beowulf cluster is a computer cluster of what are normally identical, commodity-grade computers networked into a small local area network with libraries and programs installed which allow processing to be shared among them. The result is a high-performance parallel computing cluster from inexpensive personal computer hardware.

The name Beowulf originally referred to a specific computer built in 1994 by Thomas Sterling and Donald Becker at NASA. The name "Beowulf" comes from the Old English epic poem of the same name.

    VI.                  Creating a user for running MPI jobs

There are many reasons for creating a separate user for running MPI. Following are some of the concluded reasons (ref:https://www-users.cs.york.ac.uk/~mjf/pi_cluster/src/Building_a_simple_Beowulf_cluster.html).

1.     Theres no need to remember different user names and passwords if all nodes use the same username and password.

2.     MPICH2 can use SSH for communication between nodes. Passwordless login with the use of authorized keys only works if the username matches the one set for passwordless login. You dont have to worry about this if all nodes use the same username.

3.     The NFS directory can be made accessible for the MPI users only. The MPI users all need to have the same user ID for this to work.

4.     The separate user might require special permissions.

Type following command to create a new user (ptsl).

$ sudo adduser ptsl --uid 666

Repeat this on every node(same username) with the user id.

VII.                  Installing NFS

NFS can share a local directory to every node.

You need to install nfs-kernel-server on the master node, and install nfs-common on every slave.

On master:

$ sudo apt-get install nfs-kernel-server

On slave:

$ sudo apt-get install nfs-common

We will then share the ptsl home directory on master node to every slave node.

We need to edit /etc/exports file on the master node

$ vi /etc/exports

/home/ptsl *(rw,sync,no_subtree_check)

Save and exit the file and restart NFS daemon on master node.

$ sudo service nfs-kernel-server restart

After that the directory /home/ptsl should now share with other nodes. Check this by typing the following command.

$ showmount -e master

And it should print out like this

Export list for master:

/home/ptsl *

And now we should disable ubuntu firewall system that blocks connections from other slaves to master. Type this on the master node.

$ sudo ufw allow from 192.168.1.0/24

After that we mount the directory on every slave to the master directory.

On slave1:

$ sudo mount master:/home/ptsl /home/ptsl

On slave2:

$ sudo mount master:/home/ptsl /home/ptsl

On slave3:

$ sudo mount master:/home/ptsl /home/ptsl

$ vi /etc/fatab

VIII.                  Setup passwordless SSH between each nodes

For parallel processing, communication between master and other slave nodes is important. So in order to execute a program at master node as well as other nodes we need to create a passwordless SSH between them.

First of all, install SSH on all the nodes(master as well as slave).

$ sudo apt-get install ssh

After successfully installing, we need to create a key of ssh in our /home/ptsl directory. So if we generate an ssh key at one node it will automatically have an ssh on every node (thats why we install NFS first). Use the following command to create a key by superuser.

$ su ptsl

$ ssh-keygen

It will ask to enter a passphrase, leave it empty!!!

Finally type this on the master node using the user ptsl.

$ ssh-copy-id localhost

Testing:

Now we are able to ssh to other nodes on the master node WITHOUT entering a password.

$ ssh slave1

$ echo $HOSTNAME

slave1

If it successfully connects to every node we can proceed to install mpich.

 

    IX.                  Install mpich3.3.2

Mpich is an external agents that spawn and manage parallel jobs. The version we use in this doc is mpich3.3.2

    XI.                  Parallel a job on the cluster

Create a new file called mpi_hello.c in the directory ptsl/source (create one you don't have one) and add this to this file. And create a directory call bin (storing binary files)

 

 

#include <stdio.h>

#include <mpi.h>

 

int main(int argc, char** argv) {

int myrank, nprocs;

 

MPI_Init(&argc, &argv);

MPI_Comm_size(MPI_COMM_WORLD, &nprocs);

MPI_Comm_rank(MPI_COMM_WORLD, &myrank);

 

printf("Hello from processor %d of %d\n", myrank, nprocs);

 

MPI_Finalize();

return 0;

}

And then compile the code with mpi c compiler.

$ mpicc source/mpi_hello.c -o bin/mpi_hello

Run it

$ mpiexec -n 6 -f hosts ./bin/mpi_hello

mpiexec

command to execute an mpi-compatible binary

-n 6

the number of cores to ask for - this should not be more than the sum of cores listed in the hosts. You can check how many cores one computer has with this.

$ nproc

-f ./hosts

the location of the hosts file

./mpi_hello

the name of the binary to run

 

Alternatively, you can check parallel processing using another simple example.

* -*- Mode: C; c-basic-offset:4 ; indent-tabs-mode:nil ; -*- */

/*

* (C) 2001 by Argonne National Laboratory.

* See COPYRIGHT in top-level directory.

*/

#include "mpi.h"

#include <stdio.h>

#include <math.h>

 

double f(double);

 

double f(double a)

{

return (4.0 / (1.0 + a * a));

}

 

int main(int argc, char *argv[])

{

int n, myid, numprocs, i;

double PI25DT = 3.141592653589793238462643;

double mypi, pi, h, sum, x;

double startwtime = 0.0, endwtime;

int namelen;

char processor_name[MPI_MAX_PROCESSOR_NAME];

 

MPI_Init(&argc, &argv);

MPI_Comm_size(MPI_COMM_WORLD, &numprocs);

MPI_Comm_rank(MPI_COMM_WORLD, &myid);

MPI_Get_processor_name(processor_name, &namelen);

 

fprintf(stdout, "Process %d of %d is on %s\n", myid, numprocs, processor_name);

fflush(stdout);

 

n = 10000; /* default # of rectangles */

if (myid == 0)

startwtime = MPI_Wtime();

 

MPI_Bcast(&n, 1, MPI_INT, 0, MPI_COMM_WORLD);

 

h = 1.0 / (double) n;

sum = 0.0;

/* A slightly better approach starts from large i and works back */

for (i = myid + 1; i <= n; i += numprocs) {

x = h * ((double) i - 0.5);

sum += f(x);

}

mypi = h * sum;

MPI_Reduce(&mypi, &pi, 1, MPI_DOUBLE, MPI_SUM, 0,MPI_COMM_WORLD);

 

if (myid == 0) {

endwtime = MPI_Wtime();

printf("pi is approximately %.16f, Error is %.16f\n", pi, fabs(pi - PI25DT));

printf("wall clock time = %f\n", endwtime - startwtime);

fflush(stdout);

}

MPI_Finalize();

return 0;

}
